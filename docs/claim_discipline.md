# Claim Discipline

## Supported claims

1. The project has a formal mathematical foundation.
2. The project has an executable reference implementation.
3. The implementation is benchmarked against frozen scenarios and baselines.
4. Replay is deterministic on the publication suite.
5. Durable facts have provenance coverage in the reference and oracle kernels.
6. The publication artifact is locally reproducible.

## Unsupported claims

1. Product-market readiness.
2. General-purpose natural-language memory beyond the deterministic adapter.
3. Superiority over all memory systems outside the frozen benchmark suite.
4. Commercial deployment readiness.

## Why the claims are narrow

The benchmark traces use real project language and adversarial structure, but the extraction layer is deterministic and scenario-backed.
That is appropriate for a publication artifact because it isolates the memory architecture rather than conflating it with an unbounded NLP problem.
