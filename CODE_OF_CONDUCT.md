# Code of Conduct

Use a professional standard suitable for a technical review environment.

Expected behavior:
- argue from evidence
- preserve claim discipline
- keep changes reproducible
- review ideas without personal attacks
- document uncertainty explicitly

Unacceptable behavior:
- harassment or intimidation
- misrepresenting benchmark results
- hiding regressions or weakening tests to pass a change
- presenting the local reference product as a hosted production service

Enforcement is intentionally simple: maintainers may reject changes or remove
participation when behavior compromises the integrity of the artifact.
